Documenting Code#

  1. Include your code from a file with lines emphasized

    ```{literalinclude} run_livereload.py
:emphasize-lines: 2-3
```

  2. The a above code generates the following:

    from livereload import Server, shell

if __name__ == '__main__':
    server = Server()
    server.watch('*.rst', shell('make html'), delay=1)
    server.watch('*.md', shell('make html'), delay=1)
    server.watch('*.py', shell('make html'), delay=1)
    server.watch('_static/*', shell('make html'), delay=1)
    server.watch('_templates/*', shell('make html'), delay=1)
    server.serve(root='_build/html')

  3. Documenting a Module:

    ```{eval-rst}
.. autoclass:: my_api.MyDemo:W

  :members:
```
    class my_api.MyDemo(name)#

    A great demo

    hello(greeting)#

    The canonical hello world example.

    A longer description with some RST.

    Parameters

    name – The person to say hello to.

    Returns

    The greeting

    Return type

    str

    Note

    This is visually nice! Also, it’s semantically rich.

Referencing Symbols#

  1. First example , we can us MyST extended markdown syntax:

    As we can see in [This is my_api.md](my_api.MyDemo), this is nice!

    As we can see in This is my_api.md, this is nice!

  2. Second example, we can also use role-base syntax:

    As we can see in {py:class}`my_api.MyDemo`, this is nice!

    As we can see in my_api.MyDemo, this is nice!

  3. Third example, provide your own text:

    As we can see in {py:class}`TESTING!<my_api.MyDemo`, this is nice!

    As we can see in TESTING!, this is nice!
